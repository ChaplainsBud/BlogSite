Program Design

3 Feb 2020

Programs are intentional arrangements of thought directed to both a computer and the community which will be sharing the burden of programming it.

Originally the main programming paradigm was procedural.

Programmers told the computer what to do in a sequence of directions.

This paradigm eventually broke into multiple forms, and languages can implement more than one of them.

Functional programming is an example of this.

While it is not the preferred paradigm for Java, one could limit themselves to directions placed in functions, and then calling those functions rather than repeating the code. In contrast, OOP dominates Java, perhaps because it allows one to put functions within objects, which in turn can be designed with classes. There is even greater control granted to the software developer.

Examples:

  • Procedural: int x = 5; int y = 6; x + 5
  • Functional: public int add(int x, int y) { return x + y; } add(5,6);
  • OOP: public static void main() { Math = new Math(); return Math.add(5,6); }
