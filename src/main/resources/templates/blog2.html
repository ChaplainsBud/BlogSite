GitHub Collectively

3 Feb 2020

GitHub significantly optimizes group projects.

One project can be uploaded to the Git Hub site for any number of developers to download.

The project can remain intact without any unwanted changes, if a selected few are granted permission to accept updates.

If anything is reject, comments can be posted that help the developer adapt to what is desired by those in charge.

If for any reason a certain developer does not like where the project is heading, they can fork the source code and make their own project with it.

All of the above allows for numerous stages to be saved and revisited, so that not merely the individual, but a group can secure the best quality software.

GitHub Example:

  • git push -> group leader
  • git pull -> following developer
  • git push origin master
  • If rejected, then receive feedback and adapt.
  • git push origin master
